04 · Root Cause

한국어 학습 데이터가 부족합니다

한 줄로

  • 이 AI 모델의 음성 인식 부분은 중국어와 영어로만 학습됨
  • 한국어 발음("ㄱ/ㅋ", "ㅗ/ㅜ", 받침)이 입력되면 처리 방법을 모름
  • 설정을 바꿔도 학습되지 않은 발음은 매칭 불가

6.4 · 단가가 내려가도 — 남는 본질적 한계

Paid 모델 단가가 절반으로 내려와도, 다음 세 가지는 photoreal direction의 본질적 한계입니다.
  • 입 모양 정확도 100% 불가능가장 비싼 API에서도 한국어 발음과 미세한 어긋남이 남음
    broadcast 시청자가 "조금 어색하다"고 느끼는 임계점 못 넘음
  • 제스처 다양성 부족 · "말하는 사람" 모드 한 종류가리키기 · 흔들기 · 집어들기 등 라이브 커머스 필수 제스처 안 됨
    호스트가 화면에 한 자리에 고정되어 말만 함
  • 상품 직접 핸들링 불가호스트가 상품을 들어 보이거나 사용해 보이는 인서트 불가능
    모든 상품 인서트는 별도 촬영 + 합성 필요 → 제작 공정 증가
정공법은 단가·품질·제스처·상품 핸들링 모든 축에서 한계가 있습니다.
한국 시장에는 다른 접근을 검증한 콘텐츠 사례가 있습니다 — 완벽한 해법은 아니지만 살펴볼 가치가 있습니다.

10 · 사전 렌더 + 실시간 렌더

마스코트가 한 시간 전체를 진행합니다
— 사전 렌더와 실시간 렌더의 협력으로.

  • 사전 렌더는 미리 정해진 콘텐츠 (greeting · 상품 소개 · 마무리)를 방송 전에 영상으로 만들어 둡니다.
  • 실시간 렌더는 방송 중 들어오는 댓글에 답하는 부분만 즉석에서 합성합니다.

요약

  • 마스코트는 한 시간 방송 전체에 등장 — 사람 호스트처럼 끊김 없음
  • 사전과 실시간 두 모드가 beat 단위로 번갈아 진행되며, 시청자는 그 경계를 인지하지 않음
  • 사전 렌더는 비싼 영상 AI 호출을 한 번에 끝내고, 실시간 렌더는 가벼운 컴퓨트만 사용
  • 한 시간 방송 단가 목표 ≤ ₩30,000

12 · 스크립트 생성

셀러는 URL만— 시스템이 beat 단위 스크립트를 만들고, 셀러는 한 화면에서 검토합니다.

beat이란? 방송을 구성하는 가장 작은 의미 단위.•PRE — 방송 전 미리 만들어 둔 영상 bit•RT — 방송 중 실시간으로 합성하는 bit

요약

  • 셀러가 거치는 단계 — URL 등록 → 상품 선별 → 스크립트 편집 → 승인 → 방송 (5단계는 §9.2 mini-flow 참조)
  • AI가 상품을 자동 추천 — 수백 개 상품 중 방송에 적합한 N개 (기본 10–15개)
  • 스크립트는 beat 단위로 분해되어 셀러가 검토·수정 가능
  • 한 화면에서 상품 리스트와 beat 편집기를 동시에 — 컨텍스트 전환 없음
